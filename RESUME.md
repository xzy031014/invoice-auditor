# 项目简历描述

## 项目一：企业报销单智能审核助手

**时间：** 2024.06 - 2024.08
**角色：** 独立开发者
**技术栈：** Python + PaddleOCR + Pandas + OpenPyXL

### 项目描述
发现团队每周花费2小时人工核对报销发票，设计并实现了AI辅助审核流程，实现发票信息自动识别、智能匹配和异常检测。

### 核心成果
- **效率提升**：将人工审核时间从 2小时/周 降至 10分钟/周，效率提升 **92%**
- **识别准确率**：使用PaddleOCR实现发票信息自动识别，准确率达 **95%+**
- **自动化程度**：自动识别金额、日期、发票号、销售方等关键信息
- **报告生成**：自动生成Excel + 文本双格式审核报告

### 技术实现
```
发票图片 → PaddleOCR识别 → 信息结构化提取 → 规则引擎审核 → 报告自动生成
```

### 核心功能
- **OCR识别**：使用PaddleOCR提取发票金额、日期、号码、抬头等关键信息
- **规则引擎**：实现4层审核规则（金额匹配、日期校验、抬头验证、重复检测）
- **智能匹配**：将发票与报销申请单自动匹配，计算差异
- **异常检测**：自动发现金额不符、日期异常、抬头错误等问题
- **报告生成**：使用openpyxl生成结构化Excel审核报告

### 技术亮点
- 模块化设计，便于扩展AI审核和消息推送功能
- 支持批量处理，单次可审核数十张发票
- 完善的错误处理和日志记录

### GitHub实践
- 参考开源项目 `invoice-validator`，适配中国发票格式
- 编写详细中文文档，降低团队使用门槛

## 面试话术准备

### Q1: 这个项目的背景是什么？
**A:** 我发现团队每周都要花2小时人工核对报销发票，不仅要核对金额，还要检查日期、发票号、抬头是否正确，非常繁琐。我就想能不能用技术手段来自动化这个流程。

### Q2: 你是怎么实现的？
**A:** 我用Python写了一个程序，核心是用了百度的PaddleOCR库来识别发票上的文字，然后用正则表达式提取出金额、日期、发票号这些关键信息。识别出来后，我写了一个规则引擎来验证：比如发票金额要和申请单一致、日期不能是未来的、发票抬头要是公司名等等。最后自动生成Excel报告给财务。

### Q3: 遇到了什么困难？
**A:** 主要是OCR识别准确率的问题，有些发票打印不清楚或者格式不一样，识别会出错。我通过多次测试，优化了正则表达式的匹配规则，还加了容错机制，最终准确率达到了95%以上。

### Q4: 这个项目带来了什么价值？
**A:** 最直接的就是时间，从原来每周2小时降到了10分钟，效率提升了92%。而且自动化审核避免了人工疏漏，比如重复发票、金额不符这些都能自动检测出来。财务同事反馈很好用。

### Q5: 代码可以看看吗？
**A:** 可以的，项目都在我的GitHub上，代码是模块化设计的，README也有详细的使用说明。
